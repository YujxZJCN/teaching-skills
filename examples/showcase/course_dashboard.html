Policies
Grading schemeFinal grade = weighted sum per assessment plan (quizzes: best 8 of 10 counted). No curve. Letter cutoffs: A >= 90, B 80-89, C 70-79, D 60-69, F < 60.
Late workLabs and project milestones: 3 free late days pooled across the term (max 2 on one artifact); afterwards -10% per day, up to 3 days, then not accepted. Quizzes: no make-ups — best-8-of-10 absorbs absences. Exams: make-ups only for documented emergencies, arranged with the instructor in advance where possible.
AI useThree-tier per-assessment policy (shared/ai_era_integrity.md): Tier P for A1/A3/A5/A6 — these measure unaided fluency; Tier D for A2/A4 — professional- practice tools permitted with a graded disclosure appendix. Full table with per-tier rationales in syllabus.md, section "AI use in this course".

Learner profile
Prior knowledgeStrong programmers (CS 201; most fluent in Python). Linear algebra (MATH 210) and probability (STAT 110) formally completed, but probability preparation is mixed and linear algebra is rusty for many — the W1 lab runs a refresher.
MotivationRequired core for the 3rd-year CS cohort; strong interest in ML engineering roles and graduate study (professor's report — not surveyed).
Accessibilityunknown

Known difficulties

  • Confusing training error with test error — treating low training error as model quality
  • Treating ML as API-calling: using library models without understanding their assumptions
  • Linear algebra rust: matrix calculus and decompositions need re-activation
