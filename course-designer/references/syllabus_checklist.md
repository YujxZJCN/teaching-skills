# Syllabus Completeness Checklist

Run by `syllabus_writer_agent` before its checkpoint; gaps are reported, not silently
filled. Items marked ★ commonly carry institution-specific rules — they default to
`[NEEDS PROFESSOR INPUT]` unless the professor supplied them.

## Identification
- [ ] Course code, title, term, credits, modality
- [ ] Meeting times & location / platform
- [ ] Instructor name, contact, office hours (location + how to book)
- [ ] TA information ★ (if any)
- [ ] Prerequisites stated as students would recognize them

## Learning design
- [ ] Course description in student-facing language (why take this course)
- [ ] Learning outcomes — verbatim from the confirmed passport
- [ ] Required & optional materials with editions; cost-conscious alternatives noted
- [ ] Week-by-week schedule with topics and due dates
- [ ] Statement of how class time is used (so active-learning expectations are explicit)

## Assessment
- [ ] Every assessment: what, when, weight, which outcomes it evidences
- [ ] Grading scheme & scale ★
- [ ] Late / extension policy ★
- [ ] Regrade procedure ★
- [ ] How and when feedback is returned

## Policies
- [ ] **AI-use policy — per-assignment tier table (P/D/O) with rationale** (Quality
      Gate Q1; see `shared/ai_era_integrity.md`)
- [ ] Academic integrity policy ★ (link institutional policy; don't restate sanctions)
- [ ] Attendance & participation policy, with the alternative-route note (Quality Gate U2)
- [ ] Accommodations / disability-services statement ★
- [ ] Basic-needs / wellbeing resources statement ★ (institution-specific contacts)
- [ ] Communication norms: where questions go, expected response time

## Quality pass
- [ ] Reads in the professor's voice; no AI-boilerplate phrasing
- [ ] Support-oriented tone where equivalent (advisory, I2)
- [ ] Headings & structure accessible (single H1, ordered levels, real lists — U1)
- [ ] A student could answer "what do I do this week and how am I graded" from this
      document alone
